# Project Proposal – Custom Web Vulnerability Scanner
Group Members: Carter Steiling, Han Nguyen, Michael Mui

Project Title: Custom Web Vulnerability Scanner

Project Description: Our group will design a basic scanner that checks for common web vulnerabilities (e.g. SQL injection, XSS, weak headers, or misconfigurations). We will explain how our custom web vulnerability scanner works, and then test it against a deliberately vulnerable web app. We will tie our project back to real penetration-testing methodology by comparing automated scanning versus manual testing and discussing limitations.
Problem Statement: Web applications are constantly targeted because they are exposed to the internet and often contain sensitive user data, yet many smaller organizations rely on automated scanners without fully understanding what they detect or miss. Existing scanners can be expensive, overly complex, or operate as a “black box,” which limits learning and hands-on understanding for students and entry-level security professionals. Our project aims to address this by building a simple, transparent scanner that demonstrates how common vulnerabilities like SQL injection, cross-site scripting, and insecure headers are identified. By developing our own tool, we can better understand scanning logic, detection techniques, and false positives. This project will also highlight the importance of combining automated tools with manual testing to improve overall web application security.

AI Integration: After we collect vulnerability information via our custom scanner, we will use AI to generate a detailed report, complete with risk analysis, remediation suggestions, and MITRE ATT&CK framework mappings to improve web application security.
